# benham_top

> spinning black and white. no color in the source. color in the brain.

Benham’s top and Fechner color generator. A spinning black-and-white disk with specific arc patterns produces vivid subjective colors due to temporal frequency tuning in the visual system. The stimulus contains no chromatic information — the hue is entirely generated by neural processing. Parameterize arc pattern, rotation speed, and sector width to explore the color space.

## Live Controls

| Key | Action |
|-----|--------|
| `Space` | Toggle rotation |
| `↑ / ↓` | Increase / decrease rotation speed |
| `+ / -` | Add / remove arc sectors |
| `[ / ]` | Narrow / widen arc sectors |
| `S` | Toggle strobe mode |
| `C` | Show color prediction overlay |
| `1`–`5` | Jump to a named regime |

## Running it

No build step — it's plain ES modules and WebGL2. Serve the directory with
any static file server and open it (must be served over HTTP, not opened
as a `file://` URL, since shaders are loaded with `fetch`):

```sh
npx http-server .
# or: python3 -m http.server
```

## Named Regimes

- **Benham Classic** — 3 arcs, 45° each, 5 Hz rotation
- **Maxwell** — 4 arcs, 30° each, color-segmented emergence
- **Strobe** — Variable flicker, precise temporal frequency control
- **Slow Burn** — 1 Hz, long adaptation, delayed color onset
- **Hyperspin** — 20 Hz, color disappears, pure flicker

## The Math

Fechner color mechanism (simplified):
- L-cones and M-cones have different temporal response speeds
- At flicker frequencies of 2–10 Hz, the phase lag between L and M signals creates a chromatic signal
- The color-opponent channels (red-green, blue-yellow) interpret this as hue

Sector geometry:
- Arc width: 15°–45° optimal
- Gap width: equal to or smaller than arc width
- Rotation speed: 2–10 Hz (120–600 RPM)

Subjective color predictions (typical):
- 15° arcs → blue/cyan
- 30° arcs → green/yellow
- 45° arcs → red/orange
- Individual variation is significant

## Project layout

```
index.html              entry point, canvas + overlay markup
src/js/main.js           render loop, input handling, WebGL setup
src/js/arc-pattern.js     sector geometry math (pure, unit tested)
src/js/color-predict.js   subjective color heuristic (pure, unit tested)
src/js/presets.js         the Named Regimes table as data
src/shaders/disk.vert     fullscreen-quad pass-through
src/shaders/disk.frag     per-pixel arc pattern + rotation
src/shaders/strobe.frag   full-viewport flicker overlay
examples/                 standalone snippets, see examples/README.md
test/                     node --test unit tests for the pure modules
```

See `docs/math-reference.md` for the Fechner color model in more detail
and `docs/visual-targets.md` for what each regime should look like.
